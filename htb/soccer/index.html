Uso de sqlmap

La herramienta más común para explotar SQLi es sqlmap. Sin embargo, esta vez no funcionará porque la vulnerabilidad se explota a través de WebSocket, no HTTP. Por lo tanto, debemos usar un proxy para traducir las peticiones HTTP a WebSocket y viceversa, de modo que sqlmap ataque al proxy y el proxy reenvíe las peticiones HTTP al servidor de WebSocket.

Buscando un proxy así, podemos encontrar este artículo, que muestra una implementación en Python.

Solo necesitamos editar esta línea:

ws_server = "ws://soc-player.soccer.htb:9091"

El payload en JSON es correcto, por lo que ahora podemos iniciar el servidor proxy:

$ python3 proxy.py
