Se nos proporciona un código en Python para cifrar la flag, y el correspondiente archivo out.txt:

from Crypto.Util.number import getPrime, isPrime, bytes_to_long  
from Crypto.Util.Padding import pad
import os


FLAG = b"HTB{--REDACTED--}"


def getPrimes(coefficient):
    while True:
        a = getPrime(512)
        p = 3 * coefficient * a + 2
        if isPrime(p):
            break
    while True:
        b = getPrime(512)
        q = 3 * coefficient * b + 2
        if isPrime(q):
            break
    return p, q


def encrypt(message, coefficient):
    p, q = getPrimes(coefficient)
    n = p * q

    padded_message = bytes_to_long(pad(message, 256))
    message = bytes_to_long(message)

    c1 = (message * (message + coefficient)) % n
    c2 = (padded_message * (padded_message + coefficient)) % n
    return (n, c1, c2)


def main():
    coefficient = getPrime(128)
    out = ""

    message = FLAG[0:len(FLAG)//2]
    n1, c1, c2 = encrypt(message, coefficient)
    out += f"{n1}\n{c1}\n{c2}\n"

    message = FLAG[len(FLAG)//2:]
    n2, c3, c4 = encrypt(message, coefficient)
    out += f"{n2}\n{c3}\n{c4}\n"

    out += f"{len(FLAG)}"

    with open("out.txt", "w") as f:
        f.write(out)


if __name__ == '__main__':
    main()

En primer lugar, vemos que el programa coge un número primo de 128 bits llamado coefficient para generar otros números primos $p$ y $q$. Sea $C$ el valor de coefficient, entonces los números primos $p$ y $q$ se calculan como:

$$ p = 3 a C + 2 $$

$$ q = 3 b C + 2 $$

Donde $a$ y $b$ son números primos de 512 bits. Usando este método extraño, tenemos dos módulos: $n_1$ y $n_2$. Por tanto, seremos capaces de encontrar el valor de $C$ porque es común a ambos módulos. Sean $p_i = 3 a_i C + 2$ y $q_i = 3 b_i C + 2$, entonces:

$$ \begin{equation} \begin{split} n_i &= p_i \cdot q_i \\ &= (3 a_i C + 2) \cdot (3 b_i C + 2) \\ &= 9 a_i b_i C^2 + 6 a_i C + 6 b_i C + 4 \end{split} \end{equation} $$

Nótese que:

$$ \begin{equation} \begin{split} n_i - 4 &= 9 a_i b_i C^2 + 6 a_i C + 6 b_i C \\ &= 3 C \cdot (3 a_i b_i C + 2 a_i + 2 b_i) \end{split} \end{equation} $$

Como tenemos $n_1$ y $n_2$, entonces $3 C = \gcd{(n_1 - 4, n_2 - 4)}$, por lo que

$$ C = \frac{\gcd{(n_1 - 4, n_2 - 4)}}{3} $$

Genial, ahora vamos a ver cómo se calculan los textos cifrados $c_1$ y $c_2$ (el procedimiento es el mismo para $c_3$ y $c_4$). Sea $m_1$ la primera mitad de la flag:

$$ \begin{equation} \begin{split} c_1 &\equiv m_1 \cdot (m_1 + C) \pmod{n_1} \\ &\equiv m_1^2 + m_1 C \pmod{n_1} \end{split} \end{equation} $$

$$ \begin{equation} \begin{split} c_2 &\equiv \mathrm{pad}\, (m_1, 256) \cdot \left(\mathrm{pad}\, (m_1, 256) + \cdot C\right) \pmod{n_1} \\ &\equiv \mathrm{pad}\, (m_1, 256)^2 + \mathrm{pad}\, (m_1, 256) \cdot C \pmod{n_1} \end{split} \end{equation} $$

La función pad añade más bytes al mensaje de manera predecible (PKCS7). Por ejemplo, si un mensaje de 12 bytes se rellena hasta 16 bytes, el relleno serán 4 bytes \x04. Aquí hay algunos ejemplos:

$ python3 -q
>>> from Crypto.Util.Padding import pad
>>> pad(b'A' * 12, 16)
b'AAAAAAAAAAAA\x04\x04\x04\x04'
>>> pad(b'A' * 10, 16)
b'AAAAAAAAAA\x06\x06\x06\x06\x06\x06'
>>> pad(b'A' * 1, 16)
b'A\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f\x0f'

Podemos expresar la función pad en términos matemáticos como sigue, siendo $L$ la longitud del mensaje:

$$ \mathrm{pad}\,(m_i, 256) = m_i \cdot 256 ^ {256 - L} + P $$

Y $P$ (el relleno) sigue la siguiente expresión. Por motivos de simplificación, sea $K = 256 - L$:

$$ P = K \cdot (1 + 256 + 256^2 + \dots + 256^{K - 1}) $$

Que es la suma de una progressión geométrica, que equivale a

$$ P = K \cdot \frac{256^K - 1}{256 - 1} $$

Por tanto, podemos expresar $c_2$ como

$$ \begin{equation} \begin{split} c_2 &\equiv \mathrm{pad}\, (m_1, 256)^2 + \mathrm{pad}\, (m_1, 256) \cdot C \pmod{n_1} \\ &\equiv \left(m_1 \cdot 256 ^ K + P\right)^2 + \left(m_1 \cdot 256^K + P\right) \cdot C \pmod{n_1} \\ &\equiv m_1^2 \cdot 256 ^{2 K} + 2 m_1 P \cdot 256^K + P^2 + m_1 C \cdot 256^K + P C \pmod{n_1} \end{split} \end{equation} $$

Ahora podemos mover algunos términos:

$$ c_2 - P^2 - P C \equiv m_1^2 \cdot 256 ^{2 K} + 2 m_1 P \cdot 256^K + m_1 C \cdot 256^K \pmod{n_1} $$

$$ c_2 - P^2 - P C \equiv 256^K \cdot (m_1^2 \cdot 256^K + 2 m_1 P + m_1 C) \pmod{n_1} $$

$$ (c_2 - P^2 - P C) \cdot \left(256^K\right)^{-1} \equiv m_1^2 \cdot 256^K + 2 m_1 P + m_1 C \pmod{n_1} $$

Vamos a definir $X$ como

$$ \begin{equation} \begin{split} X &:= (c_2 - P^2 - P C) \cdot \left(256^K\right)^{-1} \pmod{n_1} \\ &\equiv m_1^2 \cdot 256^K + 2 m_1 P + m_1 C \pmod{n_1} \end{split} \end{equation} $$

Podemos olvidarnos de $m_1^2 \cdot 256^K$ si restamos $c_1 \cdot 256^K$:

$$ \begin{equation} \begin{split} X &- c_1 \cdot 256^K \equiv \\ &\equiv (m_1^2 \cdot 256^K + 2 m_1 P + m_1 C) - c_1 \cdot 256^K \pmod{n_1} \\ &\equiv (m_1^2 \cdot 256^K + 2 m_1 P + m_1 C) - (m_1^2 + m_1 C) \cdot 256^K \pmod{n_1} \\ &\equiv 2 m_1 P + m_1 C - m_1 C \cdot 256^K \pmod{n_1} \\ &\equiv m_1 \cdot (2 P + C - C \cdot 256^K) \pmod{n_1} \end{split} \end{equation} $$

En este punto, podemos despejar $m_1$:

$$ m_1 = (X - c_1 \cdot 256^K) \cdot (2 P + C - C \cdot 256^K)^{-1} \pmod{n_1} $$

Y ya tenemos todo listo, ahora solo tenemos que programarlo. Este es un script en Python que soluciona el reto:

#!/usr/bin/env python3

import math


def main():
    with open('out.txt') as f:
        n1, c1, c2, n2, c3, c4, L = map(int, f.read().splitlines())

    C = math.gcd(n1 - 4, n2 - 4) // 3

    K = 256 - math.floor(L / 2)
    P = int(hex(K)[2:] * K, 16)

    X = (c2 - P ** 2 - P * C) * pow(256 ** K, -1, n1) % n1
    m1 = (X - 256 ** K * c1) * pow(2 * P + C - C * 256 ** K, -1, n1) % n1  

    K = 256 - math.ceil(L / 2)
    P = int(hex(K)[2:] * K, 16)

    Y = (c4 - P ** 2 - P * C) * pow(256 ** K, -1, n2) % n2
    m2 = (Y - 256 ** K * c3) * pow(2 * P + C - C * 256 ** K, -1, n2) % n2

    print(bytes.fromhex(hex(m1)[2:] + hex(m2)[2:]))


if __name__ == '__main__':
    main()
