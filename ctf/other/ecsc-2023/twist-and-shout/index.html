Por lo tanto, lo que tenemos es que, dado un punto $P$, el servidor calcula $Q = d \cdot P$. Nuestro objetivo es encontrar $d$ y, por tanto, necesitamos resolver el problema del logaritmo discreto de curva el√≠ptica (ECDLP), que es dif√≠cil ya que el orden de la curva es un n√∫mero primo.

Obs√©rvese que el servidor utiliza un algoritmo no convencional para sumar dos puntos, usando solo las coordenadas $\mathrm{x}$ de los puntos:

# Curve addition law using only x-coordinates, coded from
# https://www.hyperelliptic.org/EFD/g1p/data/shortw/xz/ladder/ladd-2002-it
def xDBLADD(P, Q, PQ):
    (X1, Z1), (X2, Z2), (X3, Z3) = PQ, P, Q
    X4 = (X2**2 - a * Z2**2) ** 2 - 8 * b * X2 * Z2**3
    Z4 = 4 * (X2 * Z2 * (X2**2 + a * Z2**2) + b * Z2**4)
    X5 = Z1 * ((X2 * X3 - a * Z2 * Z3) ** 2 - 4 * b * Z2 * Z3 * (X2 * Z3 + X3 * Z2))  
    Z5 = X1 * (X2 * Z3 - X3 * Z2) ** 2
    X4, Z4, X5, Z5 = (c % q for c in (X4, Z4, X5, Z5))

# Montgomery Ladder for scalar multiplication
def xMUL(P, k):

    for i in reversed(range(k.bit_length() + 1)):  
        if k >> i & 1:

        Q, R = xDBLADD(Q, R, P)
        if k >> i & 1:

    return Q

Con esta configuraci√≥n, que el servidor no verifica la existencia del punto, pero aun as√≠ calcula la multiplicaci√≥n, podemos intentar realizar un ataque de curva inv√°lida. Por lo general, los ataques de curva inv√°lida aprovechan el hecho de que la suma y la multiplicaci√≥n no dependen del par√°metro $b$ de la curva expresada en forma de Weierstrass ($y^2 = x^3 + a x + b$).
