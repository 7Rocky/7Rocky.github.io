USB Ripper


We are told that a sysadmin has been dumping all USB events on his Linux machine, but one day, some criminals enter another USB to steal information.

We are given two files:

$ unzip -l USB\ Ripper.zip

On the syslog file we can see that there is information about the Product, the Manufacturer and the SerialNumber of every USB device that was plugged in.

The auth.json file seems to have these identifiers separated in a JSON format. We can verify it:

$ grep ': \[' auth.json  
    "manufact": [
    "prod": [
    "serial": [

The challenge says:

Note: once you find it, “crack” it.

Maybe there is an identifier that is in syslog but not in auth.json. We can use some shell scripting to take all identifiers:

$ grep -ioE '[0-9A-F]{16,}' syslog | head    
180038A2EF71F12960D35319D3EF153

$ grep -ioE '[0-9A-F]{16,}' syslog | wc -l
  300000

$ grep -ioE '[0-9A-F]{16,}' auth.json | wc -l  
  300000

Now we can sort the identifiers and output them into a file:

$ grep -ioE '[0-9A-F]{16,}' syslog | sort -u > syslog_hashes

$ grep -ioE '[0-9A-F]{16,}' syslog | sort -u > auth_json_hashes

And finally, check their differences:

$ diff syslog_hashes auth_json_hashes  
0a1
> 0C09754B3AB54EC3581DA83B287685B2
206480d206480
< 71DF5A33EFFDEA5B1882C9FBDC1240C6

We have two candidates. These are MD5 hashes, so we can try to crack them using john (John the Ripper):

$ echo 71DF5A33EFFDEA5B1882C9FBDC1240C6 > hash

$ john --wordlist=$WORDLISTS/rockyou.txt --format=Raw-MD5 hash
